## 📋 9 СТЪПКИ — ПО РЕД, БЕЗ СКИП

Шеф чатът изпълнява **по една стъпка наведнъж**, чака потвърждение, продължава.

---

### СТЪПКА 1 — RECONCILIATION (Rule #8)

Шеф чатът минава целия chat **3 пъти** (Закон №3) и извежда:

**PLANNED:** какво беше планирано за днес  
**ACHIEVED:** какво реално е свършено (списък на commits + decisions)  
**DELTA:** какво не е свършено и защо  
**ROOT CAUSES:** ако нещо се отбяга — защо  
**LESSONS:** какво научихме днес  
**TOMORROW:** какво е приоритет утре

Формат:

```markdown
## Reconciliation [дата]

### PLANNED
- [Задача 1]
- [Задача 2]

### ACHIEVED
- ✅ [Задача 1] — commit [hash]
- ⚠️ [Задача 2] — частично, [причина]

### DELTA
- ❌ [Задача 3] — не е почната, [причина]

### ROOT CAUSES (ако има delta)
- [Причина 1]: [обяснение]

### LESSONS
- [Какво научихме]

### TOMORROW (P0/P1/P2)
- P0: [задача]
- P1: [задача]
- P2: [задача]
```

Шеф чат показва това на Тихол → Тихол одобрява или коригира → Шеф продължава.

---

### СТЪПКА 2 — FETCH VERIFICATION (НОВО в v3.0)

Шеф чатът проверява синхронизация между:
- Droplet (`/var/www/runmystore/`)
- GitHub repo (`tiholenev-tech/runmystore`)
- Project knowledge (този чат)

```bash
cd /var/www/runmystore
git fetch origin main
git status
git log origin/main..HEAD --oneline   # local commits not pushed
git log HEAD..origin/main --oneline   # remote commits not pulled
```

Ако има несинхронизирани файлове:
- Шеф чатът пише списъка
- Дава точна команда за поправка
- Тихол изпълнява

Ако всичко е синхронизирано → продължава към стъпка 3.

### СТЪПКА 3 — UPDATE STATE + COMPASS

Шеф чатът пише draft на:
- `STATE_OF_THE_PROJECT.md` (текущо състояние)
- `MASTER_COMPASS_UPDATE.md` (само diff/append блок, не цял файл)

Съдържание:
- Нов LOGIC CHANGE LOG entry за днешната дата

- Phase progress update (ако има)

**Drafts в `/tmp/`, не direct write в `/var/www/`.**

Тихол преглежда → одобрява → продължава.

## ✅ ПРОВЕРКА ЧЕ ПРОТОКОЛЪТ Е РАБОТИЛ

В края на правилен EOD трябва:
1. Тихол не е писал нищо освен „изпълни протокол", „свалих", „качих", „готово"
2. Има 1 commit на GitHub с обновените файлове
3. STRESS_BOARD има инструкция за нощта
4. PRIORITY_TOMORROW е готов за утре сутрин
5. Шеф чатът е казал „Благодаря ти, че бяхме заедно"

Ако някоя точка не е изпълнена → протоколът е нарушен.
