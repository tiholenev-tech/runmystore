# 🌙 END_OF_DAY_PROTOCOL.md

**Версия:** 1.0 (01.05.2026)  
**Цел:** В края на всеки шеф-чат — едни и същи стъпки, без обсъждания.

---

## 🎯 ЕДНА ФРАЗА ОТ ТИХОЛ

В края на деня Тихол пише на шеф-чата **една от двете фрази**:

**Дълъг ден:**
> "край на деня"

**Бърз приключване:**
> "приключи"

И двете значат същото — стартирай протокола.

---

## 📋 КАКВО ПРАВИ ШЕФ-ЧАТЪТ (АВТОМАТИЧНО, БЕЗ ВЪПРОСИ)

### Стъпка 1: DOCUMENT PROTOCOL
Минава целия chat три пъти. Изважда:
- Всички commits направени днес (hashes + messages)
- Всички решения на Тихол (продуктови + архитектурни)
- Всички flags / open questions
- Всички technical findings
- Всички broken promises (Sprint claims които не са реални)

### Стъпка 2: ГЕНЕРИРА 2 ФАЙЛА
**Файл 1:** `PRIORITY_TODAY.md` (за следващия ден)
- Стартов протокол (su - tihol → tmux → claude)
- Browser test (ако има нови commits за тестване)
- P0 / P1 / P2 задачи
- Open flags
- Статус таблица с днешните commits

**Файл 2:** `MASTER_COMPASS_UPDATE.md` (только diff/append блок, не цял файл)
Съдържание:
- Нов LOGIC CHANGE LOG entry за днешната дата
- Нови REWORK QUEUE entries (всички flag-нати днес)
- Phase progress update (ако има)
- Update на специфични секции (DESIGN-KIT version, и т.н.)

### Стъпка 3: ДАВА И ДВАТА ФАЙЛА КАТО DOWNLOAD
Шефът използва `present_files` tool → Тихол виждa download links.

### Стъпка 4: ПИШЕ ИНСТРУКЦИЯ ЗА КАЧВАНЕ
Точно тези 3 стъпки, винаги еднакви:

```
1. Сваляш PRIORITY_TODAY.md → качваш в GitHub Web UI:
   https://github.com/tiholenev-tech/runmystore
   → upload PRIORITY_TODAY.md → "Replace existing file"
   → commit message: "[Date]: PRIORITY_TODAY update"

2. Сваляш MASTER_COMPASS_UPDATE.md → отваряш MASTER_COMPASS.md в repo:
   → click pencil icon (edit)
   → append съдържанието на MASTER_COMPASS_UPDATE.md в края
   → commit message: "[Date]: COMPASS LOGIC LOG + REWORK QUEUE entries"

3. Пишеш на шеф-чата: "качено" → готово.
```

### Стъпка 5: ПОТВЪРЖДАВА И ЗАТВАРЯ
Шефът чака "качено" от Тихол.
Ако Тихол каже "качено" → "Лек край на деня."
Ако Тихол попита нещо → отговаря, продължава.
Ако Тихол НЕ отговори → не настоява, не повтаря.

## ❌ КАКВО ШЕФ-ЧАТЪТ НЕ ПРАВИ

- НЕ пита Тихол въпроси преди да започне (само ако има CRITICAL gap в данните)
- НЕ обсъжда дали да включи нещо или не — включва всичко важно по преценка
- НЕ дава файла за обсъждане ("какво мислиш?") — финален е
- НЕ пише огромни обяснения — само файлове + 3-стъпкова инструкция
- НЕ повтаря "минах разговора три пъти" освен в ПРОТОКОЛ ИЗПЪЛНЕН ред в края на файла

## 🚨 EDGE CASES

### А) Тихол излиза рано без "край на деня"
Ако chat-ът свърши без думата → шеф-чатът пише `STATE_OF_DAY_[date].md` (като 01.05) и качва само него. Не претендира за пълен end-of-day.

### Б) Малко работа през деня (1-2 commits)
Skip MASTER_COMPASS_UPDATE.md ако няма реални logic changes. Само PRIORITY_TODAY.md.

### В) Тихол casно пита "обнови ли документите"
Шефът проверява статус — ако са качени (Тихол е казал "качено"), казва ДА. Ако не — казва "Не. Чакам ти да приключим деня с 'край на деня'."

### Г) Code Code още работи в края на деня
Шефът завършва end-of-day без да чака. Bъв файла отбелязва: "[N] активни Code Code сесии при затваряне — verify утре сутрин."
