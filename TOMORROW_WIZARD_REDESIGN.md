### B. AI генерира ВСИЧКО в едно обаждане

От снимката AI връща наведнъж (БЕЗПЛАТНО — едно обаждане):

## 🤖 AI ОБУЧЕНИЕ — НИВО 1 + 2 ВСИЧКИ ТУК (S144)

### Защо в S144 (не S143):

Ниво 1 и 2 имат смисъл САМО когато AI обработва снимки в wizard. Сега в S143 правим търсачка — не AI обаждания. Логично е да са заедно с wizard.

### Ниво 1 — Баркод lookup (НАЙ-ПРОСТО)

**Преди да направим AI обаждане за снимка:**

```php
$existing = DB::run("SELECT * FROM products WHERE barcode=? AND tenant_id=? LIMIT 1",
    [$barcode, $tenant_id])->fetch();

if ($existing) {
    // Копирай всичко — AI НЕ се обажда. Спестено ~€0.03.
    return copyFromExisting($existing);
}
```

**Допълнителна логика:** ако баркодът съществува в **друг tenant** (друг магазин на платформата) → може да копираме нечувствителни полета (категория, цвят, описание, материя). НЕ копираме цени, маржове, доставчик.

### Ниво 2 — Perceptual hash + ai_snapshots таблица

**Стъпка А:** Нова таблица за snapshot-и:

```sql
CREATE TABLE ai_snapshots (
    id INT AUTO_INCREMENT PRIMARY KEY,
    tenant_id INT NOT NULL,
    image_hash VARCHAR(64) NOT NULL,
    ai_response JSON NOT NULL,
    confidence FLOAT,
    confirmed_by_user TINYINT DEFAULT 0,
    final_data JSON,
    product_id INT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_hash (image_hash),
    INDEX idx_tenant_confirmed (tenant_id, confirmed_by_user)
);
```

**Стъпка Б:** Преди ново AI обаждане → провери за подобни:

```php
$newHash = computePHash($photo);
$similar = DB::run("
    SELECT ai_response, final_data FROM ai_snapshots
    WHERE tenant_id=? AND confirmed_by_user=1
    AND BIT_COUNT(CAST(image_hash AS UNSIGNED) ^ CAST(? AS UNSIGNED)) < 8
    ORDER BY created_at DESC LIMIT 5
", [$tenant_id, $newHash])->fetchAll();

if ($similar) return reuseResponse($similar[0]);  // €0
$response = callGeminiAI($photo);  // €0.03
saveSnapshot($newHash, $response);
return $response;
```

**Очаквана икономия:** ~50% спестявания след 1000+ артикула с потвърждения.

### Ниво 3, 4 — ОТЛОЖЕНИ!

| Ниво | Кога | Защо отложен |
|---|---|---|
| 3 (rule engine) | След 2-3 месеца | Изисква 1000+ потвърдени артикули |
| 4 (локален ML модел) | Чак при 50+ магазина | Инфраструктура €150/мес надделява над спестяванията |
