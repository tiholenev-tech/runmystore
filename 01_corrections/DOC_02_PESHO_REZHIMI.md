# 1. ЗАЩО ИМА ДВА РЕЖИМА

RunMyStore има две противоположни аудитории. **Пешо** (seller, non-tech owner) иска един интерфейс без менюта, AI да му казва какво да прави. **Owner/Manager** (tech-savvy) иска пълен контрол, dashboards, таблици, модули.

Опитахме една версия за двамата — не работи. Пешо се плаши от complexity, Owner се отегчава от simplicity. **Два режима, една кодова база.**

Модулите са едни и същи (products.php, sale.php, orders.php). Режимите се различават в **как се достигат тези модули** — през menu (detailed) или през AI разговор (simple).

# 2. ПЕШО — ПОДРОБНАТА БИОГРАФИЯ

## 2.1 Демография

- **Възраст:** 55-65
- **Образование:** средно, техникум
- **Езици:** само БГ (някои английски думи — „бранд", „стил", „размер")
- **Живее:** малък/среден град в БГ (Плевен, Враца, Стара Загора), или крайградски квартал

## 2.2 Бизнес

- **Опит:** 20+ години в търговията (често наследен)
- **Тип:** дрехи, обувки, аксесоари, бельо, спортни
- **Размер:** 1-3 магазина, 50-80 m²
- **Оборот:** 15,000 - 80,000 лв/месец
- **Артикули:** 500-3000 SKU
- **Служители:** 1-3 (често жени на средна възраст, роднини)
- **Доставчици:** 3-10, главно Турция или Полша

## 2.3 Технологично ниво

- **Смартфон:** iPhone SE (стар) или среден Android
- **Лаптоп:** не притежава или не ползва активно
- **Facebook:** активен, лични неща
- **Viber:** главен канал с доставчици и служители
- **Email:** има, рядко чете
- **Excel:** пробвал, не разбира
- **Складова програма:** пробвал 1-2, отказал се

**Ключово:** Пешо не чете инструкции. Ако не е очевидно след 10 секунди — напуска. Затова Simple Mode няма help бутон — AI води.

## 2.4 Психологически профил

- **Loss aversion > Gain seeking** — предпочита да не загуби €100 пред да спечели €200
- **Доверие в опита си** — „Аз знам какво искат клиентите"
- **Съмнение в технологията** — „Преди работеше и без това"
- **Любопитство при видим резултат** — „Покажи ми нещо ново" → ако работи → приема
- **Социално чувствителен** — „Какво ще кажат хората"
- **Pragmatic** — не иска „cool features", иска „работи ли"

## 2.5 Типичен ден

```
07:30  Става, кафе
08:30  Пристига в магазина
09:00  Първи клиенти
09-13  Продажба + доставки
13-14  Обяд (sandwich)
14-18  Пик, продавачка помага
18:30  Затваря, брои касата
19:00  Прибира се
20-22  Вечеря
22-00  Viber, Facebook
```

**Кога ползва RunMyStore?**
- Сутрин (08:30-09:00) — „какво се случи снощи"
- Между клиенти — voice query, 30 сек
- Вечер (18:30-19:00) — „колко оборот"
- Преди лягане (22:30) — evening wrap push

**4 micro-moment-а на ден. Всеки < 2 минути.** Simple Mode е оптимизиран за кратки интеракции.

## 2.6 Цитати от реални собственици

- *„Аз съм от старата школа. Записвам на тетрадка."*
- *„Защо ми трябва компютър? 20 години работя без него."*
- *„Синът ми ми казва 'модернизирай се' — аз му казвам 'ти си модерен'."*
- *„Знам си стоката. Коя къде е."*
- *„AI-ят ще ми казва какво да правя? Аз знам по-добре."*

**После, след резултат:**
- *„Мамка му, каза ми че Nike 42 свърши, аз мислех че имам 3..."*
- *„Откъде знаеш че Мария продава повече когато е сама?"*
- *„Този твоя AI прав излезе."*

# 3. SIMPLE MODE — AI ЧАТЪТ Е ЦЕЛИЯТ ИНТЕРФЕЙС

Това е **фундаменталната промяна** от старата архитектура. AI чатът **е** Simple Mode. Целият продукт се преминава през разговор.

## 3.1 Какво вижда Пешо при отваряне

```

├─────────────────────────────────────┤
│                                     │

│  │ Добро утро! Днес имаш:      │    │

│  │ • Passionata +35% тази седм.│    │

│  │ Какво искаш да правим?      │    │
│  └─────────────────────────────┘    │
│                                     │
```

**Няма 4 бутона. Няма home екран. Няма модули.** Има AI чат, предложения от често задавани въпроси, и voice input.

## 3.2 Как Пешо „отива" в модул

**Не отива.** Пешо говори, AI pull-ва данни от модула, показва в чата.

### Примери

**Пешо: „Искам да продам"**
```
AI: Ок, какво продаваш?
Пешо: Две черни тениски L за 40 лева
AI: Две тениски за 80 лв. Потвърждаваш?
Пешо: Да
AI: ✓ Продадено. Днес: 847 + 80 = 927 лв.
```

**Зад кулисите:** AI извика `sale.create` action → създаде запис в DB → върна confirmation. Без Пешо да отваря sale.php.

**Пешо: „Колко Nike имам"**
```
AI: Nike: 12 броя общо. Размери:
    • Nike 41: 3 броя
    • Nike 42: 0 (свърши!)
    • Nike 43: 5 броя
    • Nike 44: 4 броя

    [🛒 Поръчай Nike 42] [📊 Виж продажбите]
```

**Зад кулисите:** AI извика `products.get_stock` с query „Nike" → получи числа → форматира отговор.

**Пешо: „Искам да видя списък със всички артикули на нула"**
```
AI: 7 артикула на нула:
    1. Nike Air Max 42
    2. Adidas Ultraboost 41
    3. Puma Velocity 40
    [... още 4]

    Заедно губиш ~€127/ден.

    [📦 Поръчай всичко] [Виж едно по едно]
```

**Зад кулисите:** AI извика `products.get_zero_stock` → форматира list → добави Loss изчисление.

## 3.3 Shortcut chips (не главни бутони)

Под AI bubble-а има малки **shortcut chips** — бързи команди които често се използват:

```
[🛒 Продажба] [📦 Поръчка] [🚚 Доставка] [📊 Стоката ми]
```

Това **не са главни бутони**. Това са **shortcuts към често задавани AI команди**. Тап на [🛒 Продажба] е еквивалентно на Пешо да каже „искам да продам".

Разликата е фундаментална:
- **Бутон:** отваря модул (sale.php) като отделен екран
- **Shortcut chip:** изпраща команда към AI, AI отговаря в чата

## 3.4 Често задавани въпроси (ЧЗВ)

Под shortcut chips се показват **3 предложения** за въпроси според контекста:

**Сутрин:**
- Колко продадох вчера?
- Какво трябва да поръчам?
- Какво свърши?

**Обяд:**
- Колко продадох тази сутрин?
- Кой продава днес?
- Последна продажба?

**Вечер:**
- Колко оборот днес?
- Кое се продаде най-добре?
- Кой беше топ продавач?

Тап на ЧЗВ = изпраща въпроса към AI. **Работи офлайн** (за прости въпроси се използва SQL + template, без Gemini).

## 3.5 Voice input

Долу на екрана винаги има voice input bar. Тап → микрофон се отваря, Пешо говори, транскрипцията се показва в реално време (Под-закон №1A).

```
┌─────────────────────────────────────┐
│                                     │
│  "две черни тениски L за 40 лева"  │
│                                     │
│  Confidence: 87% ✓                  │
│                                     │
│  [Диктувай отново]  [Изпрати →]    │
└─────────────────────────────────────┘
```

## 3.6 Скрити модули

В Simple Mode модулите **съществуват** (products.php, sale.php, orders.php, и т.н.) — те са ядрото на системата. Но **не са видими** в менюто.

AI pull-ва данни от тях през `/ai-action.php` router. Пешо никога не отваря модул ръчно.

**Изключения:** когато AI пуска **пълен модул екран** за определена сложна операция (напр. Zone Walk за инвентаризация). Тогава модулът се отваря като overlay с бутон „← Върни в чата" отгоре. След приключване → auto-връщане в чата.

## 3.7 Какво няма в Simple Mode

- ❌ Menu bar
- ❌ Bottom navigation
- ❌ Burger menu
- ❌ Settings бутон в хедъра (само через AI — „искам да променя нещо")
- ❌ Stats dashboard
- ❌ Filter drawer-и
- ❌ Table-и с 10 колони
- ❌ Formulari с 15 полета

**Всичко се прави през разговор.**

## 4.5 6 q-секции в модулите

Detailed Mode показва пълните **6 фундаментални въпроса** (DOC 04):
- Loss (червено) — какво губя

- Gain (зелено) — какво печеля
- Gain cause (teal) — от какво
- Order (амбър) — поръчай
- Anti-order (сив) — не поръчай

В Simple Mode тези въпроси не се показват като секции — AI ги превежда в разговорни съобщения („Губиш €420/седмица от Nike 42 — свърши").

## 4.6 Stats module

В Detailed Mode има **пълен stats dashboard**:

- Cross-store comparison (ако multi-store)
- Custom date ranges
- Exports (CSV, Excel, PDF)

В Simple Mode — всичко това е през AI разговор („покажи ми продажбите от миналата седмица").

## 4.7 Kato е достъпен Detailed

- Ъгъл на Simple Mode има „← Разширен" бутон (само за owner/manager, не за seller)
- Default-ът за нови owner акаунти е Simple Mode за първите 4 месеца, после prompt „Искаш ли да видиш пълния режим?"
- Default за manager е винаги Detailed

# 5. ТРИТЕ РОЛИ

## 5.1 OWNER (собственик) — 👑

**Вижда всичко без изключение:**
- Печалби, маржове
- Доставни цени
- Одит лог
- Настройки
- Управление на потребители

- Прави поръчки
- Изпраща SMS
- Добавя потребители

**Default режим:** Simple Mode за първите 4 месеца, после избор.

## 5.2 MANAGER (управител) — 🔑

**Вижда operational, НЕ финанси:**
- ✅ Заприхождава (вижда доставни цени при доставка)
- ✅ Ценообразуване
- ✅ Продажби, трансфери, справки, връщания
- ✅ Наличности навсякъде
- ❌ НЕ вижда печалби в €
- ❌ НЕ вижда общи финансови отчети
- ❌ НЕ изтрива данни
- ❌ НЕ управлява потребители

**Default режим:** Detailed Mode винаги.

## 5.3 SELLER (продавач) — 💼

**Вижда само продажба:**
- ✅ Продажба (sale.php)
- ✅ Търсене на артикули (без доставни цени)
- ✅ Отстъпки до % зададен от owner
- ✅ Комисионна — автоматична
- ❌ НЕ заприхождава
- ❌ НЕ вижда покупни цени
- ❌ НЕ вижда справки
- ❌ НЕ вижда Life Board
- ❌ НЕ вижда други модули

**Default режим:** Simple Mode, НЕ може да превключва.

## 5.4 Multi-role insights

Един и същ insight има **различни templates** per role:

Owner може да види как изглежда UI за Manager:
- Read-only режим
- Не записва нищо
- Бързо обратно
- Полезно за testing

- Role проверка в PHP backend, не frontend
- JWT token с role claim
- Всеки API endpoint проверява role
- Audit log за всяка role change
- Не може самостоятелно upgrade (само Owner може)

# 6. 3D ACCESS MATRIX (РОЛЯ × ПЛАН × РЕЖИМ)

Всяко действие в системата се филтрира през **3 измерения**:

1. **Роля** — owner / manager / seller
2. **План** — free / start / pro
3. **Режим** — simple / detailed

## 6.1 Пример

| Действие | Owner+PRO+Detailed | Manager+START+Detailed | Seller+PRO+Simple |
|---|:---:|:---:|:---:|
| Добави продукт | ✅ | ✅ | ❌ |
| Продажба | ✅ | ✅ | ✅ |
| Промяна цена | ✅ | ✅ | ❌ |
| Изтриване продукт | ✅ | ❌ | ❌ |
| Създаване поръчка | ✅ | ✅ | ❌ |
| Stats | ✅ | ✅ (без €) | ❌ |
| AI insights с action button | ✅ | ✅ (без €) | ❌ |
| Voice команди | ✅ | ✅ | ✅ |
| AI брифинг | ✅ | ✅ | ✅ |

## 6.2 Implementation — `can()` функция

```php
function can(string $action, array $ctx): bool {
    $CAPABILITIES = [
        'product.create' => [
            'roles' => ['owner', 'manager'],
            'plans' => ['free', 'start', 'pro'],
            'modes' => ['simple', 'detailed'],
        ],
        'product.delete' => [
            'roles' => ['owner'],
            'plans' => ['pro'],
            'modes' => ['detailed'],
        ],
        'sale.create' => [
            'roles' => ['owner', 'manager', 'seller'],
            'plans' => ['free', 'start', 'pro'],
            'modes' => ['simple', 'detailed'],
        ],
        'stats.view_profit' => [
            'roles' => ['owner'],
            'plans' => ['start', 'pro'],
            'modes' => ['detailed', 'simple'],
        ],
        'ai.voice_free_text' => [
            'roles' => ['owner', 'manager', 'seller'],
            'plans' => ['pro'],
            'modes' => ['simple', 'detailed'],
        ],
    ];

    $cap = $CAPABILITIES[$action] ?? null;
    if (!$cap) return false;

    return in_array($ctx['role'], $cap['roles'])
        && in_array($ctx['plan'], $cap['plans'])
        && in_array($ctx['mode'], $cap['modes']);
}
```

## 6.3 Middleware

```php
function requireCan(string $action): void {
    global $user;
    if (!can($action, [
        'role' => $user['role'],
        'plan' => $user['plan'],
        'mode' => $user['mode'],
        'store_id' => $_SESSION['store_id'],
    ])) {
        http_response_code(403);
        exit(json_encode(['error' => 'FORBIDDEN']));
    }
}

requireCan('product.delete');
```

# 8. DO NOT DISTURB WINDOW

## 8.1 Проблемът

03:47 — Пешо хърка. Телефонът под възглавницата. Voice FAB случайно активиран. Микрофонът улавя „мъмм пратат...". STT: „продай 50 тениски". AI изпълнява.

## 8.2 Решение

```sql
ALTER TABLE tenants
  ADD COLUMN dnd_start TIME DEFAULT '22:00',
  ADD COLUMN dnd_end TIME DEFAULT '07:00',
  ADD COLUMN dnd_timezone VARCHAR(64) DEFAULT 'Europe/Sofia',
  ADD COLUMN dnd_action_policy ENUM('block','require_pin','log_only') DEFAULT 'require_pin';
```

## 8.3 Правила

В DND window:
- **block:** отказва AI actions напълно
- **require_pin:** иска 4-цифрен PIN (voice = spoken digits)
- **log_only:** log + execute

Per-tenant конфигурация в Settings.
