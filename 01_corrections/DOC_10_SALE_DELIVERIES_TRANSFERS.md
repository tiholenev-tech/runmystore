# 1. SACRED SALE — ФИЛОСОФИЯ

Sale.php е **най-критичният модул** в продукта. Не защото е technically complex — а защото:
- Клиент пред касата не чака
- Грешка в продажба = директна загуба на пари
- Seller не може да debug-ва

**Sacred Sale правила:**
1. **Never block a sale** — дори без баркод, име, каквото и да е
2. **Voice е primary** → scanner secondary → ръчен списък tertiary
3. **Response time < 100ms** за всяка операция
4. **Offline works** — Capacitor queue ако няма net
5. **AI toast async** — никога не блокира продажба

# 4. STRIPE TERMINAL INTEGRATION

За приемане на карти на физическата каса.

## 4.1 Hardware
Stripe Terminal reader (€59 one-time) — BBPos WisePad 3 или Stripe M2.

## 4.2 Flow
```
1. Клиент избира „Карта"
2. Sale.php извиква Stripe Terminal API
3. Reader показва сумата
4. Клиент tap-ва / inser-ва карта
5. Approval → sale.completed
6. Receipt options
```

## 4.3 Fees
- Stripe Terminal: 1.4% + €0.25 per transaction (EU)
- За PRO план: включено в цената
- За START: pass-through (Пешо плаща)

│                                     │

┌─────────────────────────────────────┐
│ Очаквам трансфер от магазин A       │

## 7.3 Stock movements ledger

Всеки трансфер → 2 реда в `stock_movements`:
- `transfer_out` в store A (quantity_millis=-X)
- `transfer_in` в store B (quantity_millis=+X)

Свързани с **общ operation_id**.

# 8. WOOCOMMERCE 2-WAY SYNC

## 8.1 Защо
Пешо има физически магазин + online shop. Двата трябва да споделят stock.

## 8.2 Архитектура

```
RunMyStore  ←→  WooCommerce
    │               │
    │   REST API   │
    └──────────────┘
```

**Push from RunMyStore → WooCommerce:**
- Нов артикул → create product
- Stock промяна → update inventory
- Промяна на цена → update price

**Pull from WooCommerce → RunMyStore:**
- Нова поръчка → create sale (type=online)
- Return → stock adjustment

## 8.3 Event handler

```php
EventHandlers::register('sale.completed', function($event) {
    $sale = $event['payload'];
    if ($sale['type'] === 'online') return;

    foreach ($sale['items'] as $item) {
        WooCommerce::updateStock($item['product_id'], $item['new_quantity']);
    }
});
```

## 8.4 Conflict resolution

Ako две едновременни продажби (физическа + online) опитват да продадат последния артикул:
- First-write-wins
- Second transaction → InsufficientStockException
- Notify user на second sale
