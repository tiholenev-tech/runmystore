## 6. LESSONS

1. **Always assess baseline compliance** before mockup integration — pre-commit
   hook blocks `--no-verify` workaround. Phase 1 cleanup spent ~30min, ROI was
   that all subsequent phases passed compliance smoothly.

2. **Mockup CSS comes with hardcoded values** even when claiming "uses var(--*)".
   Phase 2 tokenization found 10 hardcoded radii (8px, 10px, 999px) inside mockup
   CSS itself. Apply same px→token mapping during extraction.

3. **JS class naming drift** — existing handlers use `.sel`, `.open` while
   mockups use `.selected`, `.show`. **Solution:** dual-class toggle in JS
   wrapper (toggleFChip patched, toggleSort wrapped in s113Init).

4. **Sample data in mockups confuses dynamic vs static rendering.** Mockups had
   5× hardcoded `.prod-row` with Nike/Zara/H&M etc. — replaced with
   `<div id="prodList"></div>` for existing AJAX-driven render. lb-cards kept
   placeholder text (visual layout 1:1) but bind IDs for future JS data feeds.

5. **HANDOFF §8.2 references aren't always accurate.** `printLabel()` referenced
   as "preserved" but не съществува в codebase. Conditional check
   `if(typeof printLabel === 'function')` provides safe fallback.

6. **`design-kit/partial-header.html` ≠ `partials/header.php`.** products.php
   uses non-canonical design-kit prototype header (with palette sliders), NOT
   canonical bible-compliant header. Out of S113 scope to migrate.
