## 📁 ПРАВИЛО #1 — КАТЕГОРИИТЕ СА ДВА ВИДА

**Дата записване:** 13.05.2026 (S143)
**От:** Тихол, директно цитиран

### Двата вида категория:

**A. Глобална категория** (по умолчание, без избран доставчик)
- "Бикини" общо взето = ВСИЧКИ бикини на ВСИЧКИ доставчици
- Пример: имаш Ивон + Дафи, и двамата имат категория "Бикини"
- Натискаш само "Бикини" → виждаш бикини от Ивон + бикини от Дафи

**B. Категория по доставчик** (когато е избран доставчик)
- Първо избираш Дафи → после натискаш "Бикини"
- Виждаш САМО бикините на Дафи
- Ивон бикините са скрити

### Защо това е важно:

В магазин за дрехи **различни доставчици имат различни асортименти**. Магазинерът мисли в два режима:
1. "Какво имам общо в категория X?" → глобален преглед
2. "Какво има конкретно доставчик Y в категория X?" → за поръчка от този доставчик

### Технически: как работи в SQL

Това вече работи естествено в `advanced_search` endpoint-а (S143):

```sql
WHERE (cat избран → AND p.category_id = ?)
  AND (sup избран → AND p.supplier_id = ?)
```

Когато избереш САМО категория → филтрира само по категория (глобално)
Когато избереш категория + доставчик → филтрира по двете заедно

### UI следствие (S143 + бъдещи):

**В прозорчето с филтри:**
- Когато e избран доставчик → секцията "Категория" показва САМО категориите които ИМАТ продукти от този доставчик
- Когато няма избран доставчик → показва всички категории (глобален режим)

**Cascading dropdown logic:**
- User избира Доставчик → JS dynamic зарежда нови категории
- User изтрива Доставчика → връща се на пълен списък категории

### Пример за магазин ENI:

| Сценарий | Какво вижда |
|---|---|
| Без избор → "Бикини" | Всички 87 бикини от 5 доставчика |
| Дафи → "Бикини" | Само 23 бикини на Дафи |
| Ивон → "Бикини" | Само 31 бикини на Ивон |
| "Бикини" → после Дафи | Първо 87, после се свежда до 23 |
