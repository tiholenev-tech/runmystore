## 🔁 RE-SEED ПРОЦЕДУРА

Когато STRESS Lab tenant се счупи или иска reset:

```bash
# Стъпка 1: Backup (за всеки случай)
cd /var/www/runmystore
mysqldump -u root runmystore > /root/backup_stress_$(date +%Y%m%d_%H%M).sql

# Стъпка 2: Изтриване на всички данни на STRESS Lab tenant
python3 tools/stress/reset_stress_tenant.py

# Стъпка 3: Re-seed
python3 tools/stress/seed_stress_tenant_full.py

# Стъпка 4: Проверка
mysql -u root runmystore -e "
SELECT COUNT(*) as products FROM products WHERE tenant_id = (SELECT id FROM tenants WHERE name = 'STRESS Lab');
SELECT COUNT(*) as sales FROM sales WHERE tenant_id = ...;
"
```

**Re-seed е безопасен** — не пипа ENI tenant_id=7 или други tenants.
